# 課務 CRM 系統 — 元件互動說明文件
> **撰寫：Agent D（前端/互動）** ｜ 版本 v1.0
> 由「大區塊 → 依序往下細分元件 → 互動行為」描述。元件名稱對應實際程式。

---

## 0. 應用外殼 App（`main.jsx`）
- **職責**：session 初始化（`getSession`）、登入閘門、路由、Tweaks、Toast。
- **互動**：
  - 未登入 → 只渲染 `<LoginScreen>`。
  - 登入後 → `<Sidebar>` + `<main>`（依 `nav.view` 切模組）。
  - `goNav(view)` 切換模組並 `navTick++`（讓成效模組 remount 回第一層）。
  - 非 superuser 導到 `accounts` 會被改判回 `courses`（雙重防護）。

---

## 1. 側邊欄 Sidebar（`shell.jsx`）
```
Sidebar
├─ 品牌區（課・HBR 學院）
├─ 導覽 NAV（依角色過濾 super 項）
│   ├─ 課務：課程管理 / 期數成效管理
│   ├─ CRM：潛客管理 / 客戶管理
│   └─ 系統：帳號權限管理（僅 superuser）
└─ 使用者區（左下）
    ├─ 頭像 + 姓名 + 角色
    └─ ⚙ → 選單：顯示 Email + 「登出」
```
- **互動**：點導覽項 → `onNav(id)`；點齒輪 → 開選單；登出 → `onLogout`→`clearSession`→回登入。

---

## 2. 頂列 Topbar（`shell.jsx`）
- **組成**：麵包屑（可點返回）＋右側動作鈕＋（可選）分頁 tabs（帶數量）。
- **互動**：麵包屑點擊回上層；tabs 點擊切換 `activeTab`。各模組共用。

---

## 3. 登入畫面 LoginScreen（`auth.jsx`）
```
LoginScreen
├─ 品牌
├─ 表單卡：Email 輸入 / 密碼（眼睛切顯示）/ 錯誤列 / 記住我 / 登入鈕
└─ 示範帳號卡：admin / chen（點擊一鍵帶入）
```
- **互動**：submit→`authenticate`→失敗顯示對應錯誤；成功 `setSession`→`onLogin`。

---

## 4. 課程管理（`courses.jsx`）
```
CoursesModule
├─ Topbar：標題 + 狀態分頁 + 「匯入課程」「新增課程」
├─ ToolBar：搜尋 + 類別下拉 + 年度下拉 + 清除篩選 + 共 N 門
├─ CourseTable：列→詳情；右鍵→編輯/複製/刪除
└─ CourseDetailView（選定課程）
    ├─ Topbar 動作：新增單堂 / 新增學員 / 停課·復課 / 損益表
    ├─ KPI：學員/場次/補課截止/主講/狀態(chip)
    ├─ 場次表：點列或右鍵→SingleSessionEditModal；右鍵→取消此堂
    ├─ AddStudentsModal：StudentOrdersEditor(importOnly) 讀 BACKEND_ORDERS
    │     └─ 存 orders→updateCourse→同步 students/revenue→convertProspectsByOrders
    ├─ NewCourseModal（建課精靈，Stepper 兩步）
    │     ├─ ① 基本資訊：代碼/名稱/類別 + 招生期間 + 開課期間 + 招生追蹤設定
    │     └─ ② SessionPlanEditor：場次 + 個案（FacultyInput 串師資）
    └─ CourseSettingsModal / SessionEditModal / DeleteCourseConfirm
```
- **關鍵互動**：狀態 chip 由 `deriveCourseStatus`（日期＋停課旗標）即時推導，無手動選單；延長改編輯日期。

## 5. 期數成效管理（`performance.jsx`）
```
PerformanceModule（key=navTick 強制 remount 回第一層）
├─ PerfProductList（第一層）：商品卡（期別數/累計學員）→ onPick
├─ PerfTermList（第二層）：期別列（狀態/學員/期間/本期成交）→ onPick
└─ PerfTermDetail（第三層，Topbar 麵包屑 + 兩 tab）
    ├─ EnrollTrackTable（① 報名追蹤）
    │   ├─ KPI（池/已報名/成交/各身分數）
    │   ├─ 身分分頁 + 來源下拉 + 搜尋
    │   ├─ 工具列匯入鈕 ×3
    │   │   ├─ ProspectImportModal（搜尋/來源/狀態/全選/標記新潛客）
    │   │   ├─ CustomerImportModal（候選=購買過未報名/全選）
    │   │   └─ ExternalImportModal（手動列 + CSV 範本下載/上傳→建潛客）
    │   └─ 每列：身分chip/來源/說明會▼/報名▼/成交▼/當期備註/移除✕
    └─ SeminarPerfTable（② 說明會成效，僅個案共學會）
        └─ 每場可編輯數字 → 自動算出席率/轉換率 + 表尾本期彙總
```
- **資料源**：`recruitingPool(courseId)` 僅讀 `perf.poolMembers`（匯入驅動）；狀態覆寫存 `perf.track[key]`（key＝C:/P:/E:）。

## 6. 潛客管理（`prospects.jsx` / `prospect-modals.jsx`）
```
ProspectsModule
├─ Topbar：狀態分頁 + 匯入 + 新增潛客
├─ ToolBar：搜尋 + 來源下拉 + 排序 + 清除篩選 + 匯出

├─ 批次列（勾選後浮出）：批次修改 / 匯出所選
├─ Modals：New/Edit/Delete/Import/Batch
└─ ProspectDetailDrawer：狀態/來源(標籤)/備註(純文字) + 期別歷程時間軸(StatusTimeline)
```

## 7. 客戶管理（`customers.jsx` / `customer-modals.jsx` / `customer-export.jsx`）
```
CustomersModule

├─ ToolBar：搜尋 + 排序下拉 + 清除篩選 + 匯出

    └─ 備註（NoteEditor 標籤+文字，確認鈕→toast）
```

## 8. 帳號權限管理（`accounts.jsx`，僅 superuser）
```
AccountsModule
├─ Topbar：角色分頁 + 新增帳號
├─ 提示列：superuser 安全規則說明
├─ 列表：使用者/Email/角色chip/狀態/建立日期/操作(編輯·重設密碼·停啟·刪除)
└─ Modals：AccountFormModal(新增/編輯) / ResetPasswordModal / DeleteAccountConfirm
```
- **安全互動**：編輯/停用/刪除最後一個 superuser 或自己 → 阻擋並 toast。

## 9. 共用元件與 Token
- **共用**：`Modal / Field / Topbar / ToolBar / SearchBox / KPI / Avatar / StatusChip / Toast / Icon / StatusTimeline`。
- **跨檔共享**：掛 `window`（如 `StatusTimeline`、各 Module）或同為 classic script 之全域 const。
- **樣式**：一律 `tokens.css` 的 `var(--*)`；列表 `.tbl`、彈窗 `Modal`、空狀態置中圖示＋說明＋清除鈕。
