## 0. 系統架構總覽

- **技術**：單一 HTML 以 React 18 + Babel standalone 載入多個 `app/*.jssx`（`type="text/babel"`）。各 script 獨立作用域，跨檔共用元件掛 `window`。
- **樣式**：`app/styles/tokens.css` 的 `var(--*)` token；共用元件在 `app/shell.jsx`。
- **資料層**：全部走 `app/data.js`（種子＋localStorage 持久化＋記憶體快取）。正式版將 data.js 函式內部替換為 fetch API（介面不變）。
- **權限**：登入閘門（未登入只見登入頁）；角色 `superuser`／`staff`，僅 superuser 可見帳號權限管理。

## 1. 登入 / 帳號權限（auth.jsx / accounts.jsx）

### 1.1 登入
- Email＋密碼＋記住我；密碼可切換顯示；demo 帳號一鍵帶入。
- `authenticate(email,password)` → `{ok,user}` / `{ok:false,reason}`（notfound／disabled／badpass 各對應錯誤訊息）。
- 記住我 → localStorage，否則 sessionStorage；登入後落地課程管理。
- 登出：側欄左下頭像 → 選單「登出」→ `clearSession()`。

### 1.2 帳號權限管理（僅 superuser）
- 列表：使用者/Email/角色/狀態/建立日期＋角色分頁＋搜尋。
- 操作：新增（預設 staff）／編輯／停用·啟用／重設密碼／刪除。
- 安全規則：至少保留一個啟用中 superuser（最後一個不可停用/降權/刪除）；不可停用/刪除自己；非 superuser 側欄不顯示此項，手動導頁被導回。

## 2. 課程管理（courses.jsx / course-create.jsx / course-sessions.jsx / course-finance.jsx）

### 2.1 課程列表
- 欄位：課程訂單編號／名稱／學員人數／起訖日／狀態＋箭頭/刪除。
- 狀態分頁（全部/招生中/準備開課/上課中/已結案/停課，帶數量色點）＋類別/年度下拉＋搜尋，可並用；「N / M 門課程」＋清除篩選＋空狀態。
- 右鍵列選單：編輯／複製／刪除課程。

### 2.2 建課精靈（兩步驟，new/copy/edit 三模式）
- ① 基本資訊：代碼*、名稱*、類別*、招生期間（選填）、開課期間*；必填阻擋、代碼重複即時提示。個案共學會顯示「說明會場次」數量（預設 4，1–8）。
- ② 課程場次：場次（日期自動帶星期＋時間）×個案（主題/副標/講師/服務單位）無上限；講師串師資檔 `facultyFor(類別)`，選定自動帶服務單位。
- 本階段不設成本、不編輯學員；新建 `costs={}`/`revenue=0`/`orders=[]`；編輯原樣保留原值。

### 2.3 課程狀態（deriveCourseStatus）
- 自動：今天<招生截止＝招生中；招生截止後、未到開課日＝**準備開課**（附「還有 N 天開課」）；開課期間＝上課中；結課後＝已結案。
- 手動：停課旗標（Topbar 停課/復課），優先於日期。延長＝編輯課程改日期。

### 2.4 課程詳情
- KPI（學員/場次/補課截止/主講/狀態）＋停課紅色橫條。
- 場次表：堂次/日期/主題/主講/狀態；點列或右鍵編輯單堂、取消此堂（自動重編堂次）。
- **新增學員（後台訂單匯入）＝唯一學員來源**：讀 `BACKEND_ORDERS` 選商品帶入全部訂購人；寫 `course.orders` 並同步 `students`/`revenue`；同商品去重；觸發潛客→顧客轉換（見 §5.3）。
- 損益表匯出 .xls：營收、11 項固定成本、雜支分項、毛利、毛利率、佔比。

### 2.5 成本與營收
- 11 項固定成本＋雜支分項 `miscItems[]`；營收＝訂單加總不可手填；毛利＝營收−成本，即時計算。

## 3. 期數成效管理（performance.jsx）三層

### 3.1 第一層：商品列表
- 依 `categoryOf` 分組卡片（期別數＋累計學員）；個案共學會註記含說明會追蹤。

### 3.2 第二層：期別列表
- 欄位：期別/狀態/學員數/課程期間/**池人數/報名率/成交率**（報名率＝已報名÷池、成交率＝已成交÷池，成交率附「N 人成交」小字；池空顯示 —）。

### 3.3 第三層①：當期報名追蹤表（匯入驅動）
- 資料源 `recruitingPool(courseId)` 僅讀 `perf.poolMembers`——**課程剛建立為空**。
- 匯入入口 ×4：一鍵帶入建議名單（確認式：舊潛客全部＋舊客未報本期，可選身分）／從潛客匯入（搜尋＋來源/狀態篩選＋全選＋標記本期新潛客）／從舊客匯入（曾購買且本期未報名＋全選）／匯入外部名單（多列＋CSV 範本；同步 `createProspectIfAbsent` 建潛客）。
- **招生截止防呆**：今天 > `recruitEnd`（無則以開課日為界）→ 匯入按鈕改警示提示，不可再匯入。
- 去重：依 refId/姓名，重複略過並提示；移除成員＝確認 popup（僅移出本期池）。
- **轉換漏斗 KPI**：池內名單 → 參加說明會（僅個案共學會）→ 有意願 → 已報名；點某段篩出「停在該段」的人；隨身分分頁重算；手機版改 2 欄格狀。
- 列欄位：勾選（僅個案共學會，供批次）／姓名（可點→人員詳細抽屜）／公司職稱／身分 chip／來源／聯絡方式／說明會（**每場獨立出席紀錄** `track[key].sems={場次:狀態}`，逐場列示可移除＋新增預設已出席；場次依數字排序）／報名狀態（三態：未報名/有意願/已報名；**已報名＝成交**，無獨立成交欄）／當期備註（點擊彈窗 textarea）／移除。
- 篩選：身分分頁＋狀態下拉（未/已參加說明會、未報名/有意願/已報名）＋來源下拉＋搜尋＋漏斗段，可並用；清除篩選一併重置。
- **批次列**（勾選後浮出）：批量指定說明會場次（`BatchSeminarModal`：場次＋狀態）／**移出本期池**（確認 Modal）／清除選取。

### 3.4 第三層②：說明會成效表（僅個案共學會）
- 每場一列，**全自動彙總**：通知數＝該場被指定人數；回覆出席＝狀態≥回覆；實際出席＝已出席；成交數＝已出席且已成交。出席率＝出席÷回覆；轉換率＝成交÷出席。
- 每場可點 → **出席名單抽屜**：狀態分組（已出席/回覆出席/已通知/未出席）＋狀態 filter＋姓名搜尋＋就地改狀態（簽到）＋「新增現場出席」（同步建潛客、入池、標已出席）＋點姓名開第二層人員抽屜。
- 舊資料「已參加」（無場次）＝已出席未指定場次：計入「尚未指定場次」KPI＋警示列名。
- **歷期比較**：同商品各期 總出席/總成交/出席率/轉換率＋與上一期 ▲▼ 差異；期別下拉複選（預設最近 5 期、本期固定、全選/清除）。

### 3.5 說明會狀態流（對應實務）
「指定場次（預設已出席）→（可改）已通知 → 回覆出席（電話確認）→ 已出席（當天簽到）／未出席」；**每場獨立紀錄** `track[key].sems={場次label:狀態}`，一人可有多場紀錄（第1場未出席→第2場出席兩筆並存），改場不覆蓋舊場；舊資料 `semsOf()` 相容換算。

### 3.6 歷期比較（TermComparison）
- 同商品各期一列：總出席/總成交/出席率/轉換率＋與上一期 ▲▼ 差異；期別下拉複選（預設最近 5 期、本期固定、全選/清除）。

## 4. 潛客管理（prospects.jsx / prospect-modals.jsx）

- 欄位：潛客（姓名＋P 編號）/公司職稱/狀態 chip/來源 chips/聯絡/備註/最近聯繫/操作。
- 狀態（單選）：接通/未接通/無效/考慮/預約/婉拒/追蹤 → Topbar 分頁；來源（多選標籤）：Meta Ads/舊生推薦/Podcast/企業客戶/員工推薦/活動開發＋自訂。
- 就地編輯：狀態小選單、來源彈窗、備註彈窗（純文字）。
- 篩選：狀態分頁＋來源下拉＋搜尋（含電話正規化）＋排序（加入時間/最近聯繫）。
- 新增/修改/刪除/匯入（系統示範 6 筆真塞入/手動多列）/批次（狀態/負責人/加來源/聯繫日）/匯出（CSV/Excel）。
- **Email 唯一檢核**：新增擋下、匯入略過並提示；無 Email 不檢查。
- **電訪動線**：改狀態/來源/備註自動更新最近聯繫＝今天（存 ISO 日期）；顯示規則：今天→「今天」，其餘一律 `YYYY/M/D`（`fmtLastContact`）；詳細抽屜「今天已聯繫」鈕。
- 詳細抽屜：狀態/來源/備註＋**期別歷程時間軸**；已移除「負責人」欄（列表/批次/匯出仍保留 assignee 欄位）。

## 5. 客戶管理（customers.jsx / customer-modals.jsx / customer-export.jsx）

### 5.1 列表與篩選
- 欄位：客戶（姓名＋H 編號）/公司職稱/類型（學員/VIP）/聯絡/購買課程（2 chip＋N）/備註/最近聯繫/操作（hover 顯示）。
- 類型分頁＋搜尋（多欄＋電話正規化）＋排序（加入時間/報名次數）；全選支援中間態；批次列（批次修改/匯出所選）。

### 5.2 備註系統
- `note`（文字）＋`noteTags`（標籤）；預設池 7 個（未接/接聽/電話錯誤/無效名單/成交/婉拒/追蹤）＋自訂。
- 三處可編輯（列表彈窗/表單/抽屜），同寫同步；標籤與文字皆列入搜尋與匯出。
- 標籤池管理：自訂標籤未選取時顯示 ✕ → 從全域池刪除（＝從所有客戶移除）；預設 7 個不可刪。
- 更新備註自動更新最近聯繫＝今天；抽屜「今天已聯繫」鈕。

### 5.3 潛客→顧客轉換（data.js `convertProspectsByOrders`）
- 分界：無訂單＝潛客；有訂單＝顧客。
- 課程「新增學員」匯入訂單時逐筆比對潛客（原型：姓名＋公司；正式版：訂單編號＋帳號）→ 比中者移出潛客、建立/更新顧客（purchased 加本期課名）、本期池 deal＝已成交。
- Email 唯一檢核、匯入去重同潛客。

### 5.4 詳細抽屜與匯出
- 抽屜：基本/聯絡/購買課程（依類別分頁，超 3 堂內捲）/期別歷程時間軸/備註（確認鈕＋toast）。
- 匯出：範圍 single/selected/filtered/all；11 欄可勾選；CSV（UTF-8 BOM）或 .xls，檔名帶日期。
- 編號欄位名稱可由 Tweaks `custIdLabel` 改。

## 8. RWD 與觸控規則（tokens.css）
- 斷點：≤1024px 側欄縮窄；≤920px 側欄收 icon rail、頁籤列改横捲（`overflow-y:hidden` 防幽靈垂直捲動）、漏斗 2 欄格狀；≤700px 抽屜滿寬、Modal 貼齊視窗。
- 手機版側欄：點使用者頭像彰顯→選單（無齒輪）。
- 觸控裝置（`hover:none`/`pointer:coarse`）：列上操作鈕常駐顯示（非 hover 才現）。
- `select.input`：`appearance:none`＋自繪 caret，避免原生 ⇅ 箭頭。
- Topbar 標題與動作鈕可換行（flex-wrap），任何寬度不互壓。

## 9. 已知限制（原型階段）

1. 資料存 localStorage（單機）；正式版換 API/DB（`API契約.md`）。
2. 密碼明文比對（demo）；正式版須雜湊＋伺服器驗證。
3. 潛客轉換以姓名＋公司比對；正式版改訂單編號＋帳號。
4. 系統匯入為示範假資料；正式版接後台商品/訂單 API。
5. 編號規則（H/P/訂單編號）位數與流水起點待需求方拍板。
