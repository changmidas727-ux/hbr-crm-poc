# 課務 CRM 系統 — 使用者流程圖
> **撰寫：Agent B（UX）** ｜ 版本 v1.0 ｜ 流程以 Mermaid 描述，對應原型實作

> 在支援 Mermaid 的檢視器（GitHub、VS Code Mermaid、Typora）可直接渲染。

---

## 0. 全域進入流程（登入閘門）

```mermaid
flowchart TD
  A[開啟系統] --> B{已有有效 session?}
  B -- 否 --> C[登入畫面 LoginScreen]
  C --> D[輸入 Email + 密碼\n可勾記住我 / 點 demo 帳號帶入]
  D --> E{authenticate}
  E -- 查無帳號/已停用/密碼錯 --> C
  E -- 成功 --> F[setSession\n記住我→localStorage 否則 sessionStorage]
  B -- 是 --> G
  F --> G[主介面・落地：課程管理]
  G --> H{角色}
  H -- superuser --> I[側欄含「帳號權限管理」]
  H -- staff --> J[側欄不含帳號管理]
  G --> K[左下頭像→登出→clearSession→回登入]
```

## 1. 課程管理流程

```mermaid
flowchart TD
  A[課程管理列表] --> B[狀態分頁/類別/年度/搜尋 篩選]
  A --> C[新增課程精靈]
  C --> C1[① 基本資訊：代碼/名稱/類別\n招生期間 + 開課期間\n招生追蹤設定（說明會場次・來源標籤）]
  C1 --> C2[② 課程場次：新增場次與個案\n講師串師資檔]
  C2 --> C3[建立 → 狀態由日期自動判定]
  A --> D[點列 → 課程詳情]
  D --> D1[KPI：學員/場次/補課截止/主講/狀態]
  D --> D2[新增學員（後台訂單匯入）]
  D2 --> D2a[讀 BACKEND_ORDERS 選商品\n寫 course.orders→同步 students/revenue]
  D2a --> D2b[潛客→顧客轉換：比中者轉顧客\n本期池標記已成交]
  D --> D3[停課/復課（手動旗標）]
  D --> D4[損益表匯出 .xls]
  D --> D5[場次表：點列/右鍵編輯單堂・取消此堂]
```

## 2. 期數成效管理流程（三層）

```mermaid
flowchart TD
  A[第一層：商品/類別卡片\n期別數・累計學員] --> B[第二層：該商品期別列表\n狀態/學員/期間/本期成交]
  B --> C[第三層：期詳情（兩 tab）]
  C --> D[① 當期報名追蹤表]
  D --> D1[匯入驅動：池為空時引導]
  D1 --> D2[從潛客匯入（搜尋/來源/狀態・全選・標記新潛客）]
  D1 --> D3[從舊客匯入（曾購買未報名・全選）]
  D1 --> D4[匯入外部名單（手動/CSV範本・建潛客）]
  D --> D5[每列：身分/來源/說明會/報名/成交/當期備註/移除\n身分分頁+來源篩選+搜尋+KPI]
  C --> E[② 說明會成效表（僅個案共學會）]
  E --> E1[每場：通知/回覆/實際/成交\n自動出席率・轉換率 + 本期彙總]
```

## 3. 潛客 → 顧客 生命週期

```mermaid
flowchart LR
  P[潛客管理\n來源+狀態] -->|被匯入本期池| T[期數成效・報名追蹤]
  T -->|課程訂單匯入比中| C[轉為顧客\n移出潛客・purchased+本期課名]
  C -->|下一期再出現| O[歸「舊客」]
  T -->|未成交的新潛客| P2[下一期歸「舊潛客」]
  C -.期別歷程.-> H[客戶詳細抽屜時間軸]
  P -.期別歷程.-> H
```

## 4. 潛客管理操作流程

```mermaid
flowchart TD
  A[潛客列表] --> B[狀態分頁/來源篩選/搜尋/排序]
  A --> C[列上就地改 狀態 / 來源 / 備註]
  A --> D[新增/匯入（系統示範・手動多列）]
  A --> E[勾選 → 批次改狀態/負責人/加來源]
  A --> F[點列 → 詳細抽屜：狀態/來源/備註 + 期別歷程]
  A --> G[匯出 CSV/Excel]
```

## 5. 帳號權限管理流程（僅 superuser）

```mermaid
flowchart TD
  A[帳號權限管理列表] --> B[角色分頁/搜尋]
  A --> C[新增帳號（預設 staff）]
  A --> D[編輯（改角色/啟停）]
  A --> E[重設密碼]
  A --> F[停用/啟用]
  A --> G[刪除]
  D & F & G --> H{安全規則檢查}
  H -- 最後一個 superuser/自己 --> X[阻擋並提示]
  H -- 通過 --> Y[寫入 hbr_users_v1]
```
